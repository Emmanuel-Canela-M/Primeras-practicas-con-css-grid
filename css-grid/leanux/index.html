LEAN UX Canva

|Una vez hemos enetendido los puntos de dolor del usuario y nuestras oportunidades, es momento de ponernos en los pies del negocio y buscar desarrollar un producto minimo viable que satisfaga las necesidades del usuario aportando un valor real y que a su vez traiga ganancia al negocio.

1. Problemas

¿Que problemas tiene el producto actual y que tratamos de resolver?

  1. Resulta dificil para los usuarios saber a que dirigirse.
  2. Mal funcionamiento, no hay boton para retroceder.
  3. Tipografia pequeña.
  4. Toma mucho tiempo llenar el formulario de datos personales.

2. Soluciones

¿Que podemos crear que va a solucionar los problemas del negocio y solventar las nesecidades de los usuarios ?

  1. Reorganizar los productos y las categorias
  2. Permitirle al cliente ser ususario en la pagina para ahorrar tiempo en las ventas.
  3. Agragar navegacion donde se muestre de manera mas clara la informacion del producto.
  4. Agregar metodos de busqueda agiles.
  5. Revisar y modificar el inventario de articulos disponibles.
  6. Brindarle ayuda a los usuarios de manera que se sientan seguros al momento de facilitar la compra.

3. Resultados del negocio

¿Que problemas tiene el producto actual y que estamos tratando de resolver?

  1. Incremento de las ventas en linea
  2. Mayor reseñas.
  3. Expancion de los servicios a zonas un poco alejadas.
  4. Mayor trafico de usuarios.

4. Arquetipos de Usuario

¿En que arquetipos y usuarios nos deberiasmos enfocar primero?

  1. Personas entre 20-30 años con horarios agetreados o que viven lejos
  2. Usuarios que suelen comprar de manera frecuente en linea.

5. Hipotesis

Combinando lo que asumimos en los puntos 1, 2, 3, y 4 creamos hipotesis y planeamos la siguientes declaraciones: "Creemos que el resultado se va a obtener si el usuario logra obtiner el beneficio con prestacion"

  1. Aumento potencial del uso del servicio web
  2. Aumento de las ventas
  3. Simplificar el proceso de pago puede ahorrar tiempo y mejorar la compra rapida y eficaz

6. Resulatados y beneficios del usuario

¿Porque los usuarios utilizarian tus productos y servivios? ¿Que benefcio obtendrian de utilizarlo? ¿Que cambio en el comportamiento podremos observar que nos indique que lograron su meta?

  1. Incremento de confianza.
  2. Disminuir el nivel de frustracion por ventas.
  3. Cupones o descuentos al ser cliente fiel.
  4. Evitar hacer largas filas de espera en el local.

7. ¿Que es lo mas importante que debemos aprender ?

Para cada una de las hipotesis planteadas, identificar cuales son las que conllevan mas riesgo, cual es la que puede causar que la propuesta falle si estamos equivocados

1. Hipotesis : Aumentar las compras si mejoramos la experiencia del ususario e incorporamos nuevas funciones.


Riesgos : No cumplir con otras necesidades del ususario: interaccion interpersonal y brindar un ambiente acogedor...

2. Hipotesis : Simplificar el metodo de pago hara que las compras sean mas rapidas y eficases.


Riesgos : Saturar la aplicacion y quedarnos sin productos disponibles en corto rango de tiempo y por ende poner en riesgo la seguridad web y sobrecargar las netregas.

8. ¿Cual es la menor cantidad de trabajo que debemos hacer para aprender las siguientes cosas mas importantes?

Experimento de diseño para aprender lo mas rapido posible si la hipotesis mas arriesgada es verdadera o falsa

  1. Entrevistas y encuestas
  2. Pruebas de usabilidad
  3. Benchmarketing para evaluarnos frente a la competencia
  4. Diagrama de flujo para entender que tan largo es el proceso de compra
